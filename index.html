04 / 12 · SUNDAY · SPRINT DAY 1
Sunday is the engine.
This single day sets the creative inventory for the entire week. Nothing else competes for it.
ARCHITECTURE SET · OPEN
Set the Dossier Arc
OPEN
Open the week's Flight Dossier. Pick primary pillar + 6 supporting frameworks. Draft the Mon→Wknd narrative arc — Mission → Act I → Act II → Act III → Resolution → Next-Season Teaser. Name every day's hook angle. The week is decided before a single word is written.
BATCH CAPTURE · DEEP
Write 3 Layer 1 Scripts
DEEP
Hook · Script · On-Screen Text · Viral Caption · Pinned Comment — for each of three Jolts. One Jolt per sitting, back-to-back while in flow. Do not atomize, tag, or schedule. Pure creation only. The moment you switch to editing mode, creation output drops by ~60%.
SEED CAPTURE · CLOSE
Load Monday's Queue
CLOSE
Log 2–3 hook angles or concept seeds for Monday's sprint into the buffer queue. Brief the Content Atomizer on Sunday's scripts — it's ready to run Monday morning without a manual handoff.

RULE  ·  Sunday ends with ≥ 3 scripts queued and the week's arc locked. If neither is true, Monday's sprint absorbs the gap — not the constrained Tue–Sat blocks.

05 / 12 · MONDAY · SPRINT DAY 2
Monday locks the buffer.
By end of Monday, the queue holds 5–6 packages — enough to run 7 days at 1/day with a surplus.
BATCH CAPTURE 2 · DEEP
Write 2–3 Additional Layer 1 Scripts
DEEP
Use Sunday's seeds. Same rules — Hook · Script · On-Screen · Caption · Pinned Comment. No atomizing mid-session. No switching to review. Stay in creation mode until the last script is done. Context-switching during capture costs roughly 20 minutes of re-entry per switch.
FULL ATOMIZER RUN · AGENT

AGENT

BUFFER LOAD + RELEASE · PIPELINE
Schedule, Log, Arm
PIPELINE
Release Pipeline schedules all cleared packages: platform-correct formatting, publish-time optimization, automated release. FLY Archive entries created. Buffer count confirmed ≥ 5. The week is armed — Tuesday's first publish pulls from queue with zero creation required.

TARGET  ·  Mon closes with buffer ≥ 5 packages, all atomized and scheduled. If it closes at 4, flag it. If it closes at 3, Sunday's sprint expands next cycle. Never carry less than 3 into Tuesday.

08 / 12 · ANTIFRAGILITY LAYER
The 3-Package Buffer Protocol.
The system doesn't punish missed windows. The buffer absorbs every disruption without breaking the streak.
5–6
PACKAGES
ARMED
Full sprint output. 7 days covered + 1–2 day surplus. No action required. Sprint ran clean.
3–4
PACKAGES
NOMINAL
Minimum safe buffer. Week covered. Flag for Sunday sprint to restore surplus. System is functioning.
1–2
PACKAGES
CRITICAL
Sprint underperformed. Sunday is now mandatory full sprint. Publish from queue — zero creation in constrained blocks.
0
PACKAGES
DARK
Buffer empty. Skip publish before breaking voice. Sprint next deep day. A 1-day gap costs nothing; publishing junk costs brand equity.
BUILT-IN PERMISSION
If the buffer is ≥ 3 and you are exhausted — skip capture. Publish from queue. Rest. Reload Sunday. The system is designed for exactly this scenario. Use the permission, or the system will eventually break you, which breaks the system.

The buffer is not a productivity metric. It is a sustainability instrument. Track it every day. Not because it measures output — because it measures systemic health. A buffer trending down is a stress signal, not a failure. Respond to it early.

11 / 12 · ACTIVATION · CYCLE II
The first 12-week rollout.
Validate by hand first. Automate only after the loop is proven. Never automate a broken process.
WK 1–2 · VALIDATE
Run by hand
Execute the full Operator Loop manually. Ship 1 Dossier/week. Prove the cadence holds before touching any agents. The loop must survive contact with reality before it earns automation.
WK 3–6 · WIRE
Atomizer active
Hand Layer 1 → Content Atomizer for Layers 2–4. Measure time saved and voice drift risk. Keep the human QA gate. First Knowledge Extraction Sprint triggers if Jolt count ≥ 12.
WK 7–10 · PIPELINE
Release + Archive
Automate scheduling via Release Pipeline. Stand up the Notion Archive schema. Run the first full extraction sprint. Quarterly metrics review at week 10.
WK 11–12 · COMPOUND
Review + reset
Second extraction sprint. Full quarterly review. Lock Cycle III pillars from what the data proved — not from what felt right at the start. The data is the brief.

"Question every requirement. Remove anything unnecessary. Simplify and optimize only what remains. Automate only after the process is validated." — The only sequence that doesn't produce a well-automated broken system.
